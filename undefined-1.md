# 이 프로젝트는?

### 개발 범위

백엔드 + 프론트엔드 모든 범위 직접 개발

### 사용 대상자

나를 포함한 주변의 책 관리로 힘든 사람들 (약 3명 정도)

### 간단한 프로젝트 설명

1. 프로젝트 이름 그대로 도서 관리용 어플리케이션.
2. 회원 전용.
3. 로그인 한 회원은 자신의 로그인 정보를 이용해 책을 저장하고 조회할 수 있다.
4. 책의 모든 정보를 직접 입력하기 불편하다면, ISBN을 이용해 책 정보를 자동으로 스크레이핑 해올 수 있다.
5. 잘못 저장된 책은 수정하거나 삭제 할 수 있다.

### 만들게 된 이유

퇴사 후, 새로운 기술에 대해 독학 하면서 지금까지 구매한 책을 관리할 애플리케이션이 있으면 좋을 것 같아서 시작하게 된 프로젝트입니다. 마침, 주변에 책 관리용 어플리케이션이 있으면 좋겠다고 생각하는 사람이 몇명 더 있어서 '바로 이거다!'라는 생각이 들었습니다. 이 프로젝트에서는 지난 약 3개월 간 독학하면서 배운 내용에 대한 집대성 같은 존재라고 볼 수 있습니다.

### 진행하면서 아쉬웠던 점

새로 배우면서 적용했던 Spring Security, JWT Token같은 보안 관련 기술의 이해 부족으로 권한 분리에 문제가 있었습니다. 이 때문에 각각의 계정이 모두 단순한 일반 계정으로만 역할을 하고 관리자용 작업은 직접 서버에 접속해 작업을 해야 하는 구조가 되었습니다. 또한, OSIV(Open Session In View)에 대한 개념을 몰라 준영속 상태의 엔티티에서 지연로딩을 하려는 우를 계속 범했습니다. 다행히 이 점은 나중 OSIV에 대한 이해로 해결이 가능했습니다.

### 앞으로의 계

현재 프로젝트 진행 시 사용한 개념이나 기술의 이해의 부족으로 완성되지 못하거나 미흡했던 부분을 점검해, 해당 프로젝트의 차기 버전 구상 계획에서 좀 더 꼼꼼히 반영하고 개발할 예정입니다.
